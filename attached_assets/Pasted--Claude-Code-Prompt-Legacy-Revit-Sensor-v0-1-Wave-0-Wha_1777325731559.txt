# Claude Code Prompt — Legacy Revit Sensor v0.1 (Wave 0)

**What this is:** the kickoff prompt for the Revit-side connector. Builds a C# .NET 8 add-in for Revit 2026 that pushes a snapshot of the current model to your Replit workbench when the architect clicks a ribbon button.

**Closes the Wave 0 gate:** Empressa opens Revit, clicks "Send Snapshot," sees the project show up in the web app within seconds.

**Prerequisites:**
- Windows machine with Revit 2026 installed at `C:\Program Files\Autodesk\Revit 2026\`
- .NET 8 SDK installed (`dotnet --version` should print 8.x)
- Cursor (or VS Code) with Claude Code
- Your Replit URL + `SNAPSHOT_SECRET` from prompt 01

**How to use:**
1. Create a new empty folder, e.g. `C:\Users\cente\dev\legacy-revit-sensor\`
2. Open it in Cursor
3. Open the Claude Code panel / terminal
4. Paste everything between the `BEGIN` and `END` markers below
5. Let it build
6. Restart Revit (close any open instances first)
7. In Revit: open any project → **Add-Ins** ribbon tab → **Design Tools** panel → click **Configure** → paste Replit URL + secret → click **Send Snapshot**
8. Switch to your Replit Workbench — the snapshot should appear within 5 seconds

[Transaction(TransactionMode.ReadOnly)]
[Regeneration(RegenerationOption.Manual)]
public class SendSnapshotCommand : IExternalCommand

Execute(ExternalCommandData commandData, ref string message, ElementSet elements):
1. Load Settings via Settings.Load(). If ReplitUrl or Secret is empty, show TaskDialog "Configure first" with body "Click Configure on the Design Tools panel to set your Replit URL and secret." and return Result.Cancelled.

4. POST it: 
   try {
     var result = SnapshotClient.PostAsync(snapshot, settings).GetAwaiter().GetResult();
     // Show success TaskDialog with snapshot id and the URL of the workbench
   } catch (Exception ex) {
     // Show error TaskDialog with ex.Message
     return Result.Failed;
   }
5. Return Result.Succeeded.

Success TaskDialog:

- MainContent: "Snapshot ID: {result.Id}\nReceived: {result.ReceivedAt}\n\nOpen workbench: {settings.ReplitUrl}"
- One CommonButton: Close

Error TaskDialog:

- MainInstruction: "Could not send snapshot."

public class Settings
{
    public string ReplitUrl { get; set; } = "";
    public string Secret { get; set; } = "";

    public static string SettingsPath =>
        Path.Combine(
            Environment.GetFolderPath(Environment.SpecialFolder.ApplicationData),
            "Hauska", "DesignTools", "settings.json");

    public static Settings Load()
    {
        try
        {
            if (!File.Exists(SettingsPath)) return new Settings();
            var json = File.ReadAllText(SettingsPath);
            return JsonSerializer.Deserialize<Settings>(json) ?? new Settings();
        }
        catch { return new Settings(); }
    }

    public static void Save(Settings s)
    {
        var dir = Path.GetDirectoryName(SettingsPath);
        if (!Directory.Exists(dir)) Directory.CreateDirectory(dir);
        var json = JsonSerializer.Serialize(s, new JsonSerializerOptions { WriteIndented = true });
        File.WriteAllText(SettingsPath, json);
    }

    public bool IsValid => !string.IsNullOrWhiteSpace(ReplitUrl) && !string.IsNullOrWhiteSpace(Secret);
}

Use System.Text.Json. Trim trailing slashes from ReplitUrl on save.

SECTION 10 — Snapshot/SnapshotBuilder.cs

public static class SnapshotBuilder
{
    public static SnapshotPayload Build(Document doc) { ... }
}

Wrap everything in try/catch — if any sub-extractor throws, return an empty array for that section. The point is to get SOMETHING to Replit, not to fail because one wall is malformed.

Extract:

1. **Project info** — from doc.ProjectInformation:
   - ProjectName: pi.Name (or doc.Title if blank)
   - ProjectNumber: pi.Number
   - ClientName: pi.ClientName
   - Address: pi.Address (replace newlines with ", ")

2. **Document info**:
   - DocumentTitle: doc.Title
   - DocumentPath: doc.PathName ?? ""
   - Units: doc.GetUnits().GetFormatOptions(SpecTypeId.Length).GetUnitTypeId().TypeId.Replace("autodesk.unit.unit:", "")  — fallback to "feet" on error

3. **Active view** — from doc.ActiveView:
   - ActiveViewName: view.Name
   - ActiveViewType: view.ViewType.ToString()
   (CapturedAt: DateTime.UtcNow)

4. **Sheets** — FilteredElementCollector(doc).OfClass(typeof(ViewSheet)):
   - For each ViewSheet vs: SheetInfo(vs.SheetNumber, vs.Name, vs.GetAllPlacedViews().Count)
   - Sort by SheetNumber

5. **Levels** — OfClass(typeof(Level)):
   - For each Level lv: LevelInfo(lv.Name, lv.Elevation)
   - Sort by Elevation ascending

6. **Rooms** — OfCategory(BuiltInCategory.OST_Rooms).WhereElementIsNotElementType():
   - Cast to Room. Skip rooms where Area == 0 (unplaced).
   - For each: RoomInfo(r.Number, r.Name, r.Area, r.Level?.Name ?? "")
   - Sort by Number
   - Note: Area is already in square feet (Revit's internal unit is feet)

7. **Walls** — OfClass(typeof(Wall)).WhereElementIsNotElementType():
   - Count
   - TotalLengthFeet: sum of LocationCurve.Curve.Length for each wall (skip non-LocationCurve walls without throwing)
   - WallTypes: distinct wall.WallType.Name, sorted, max 20 entries

8. **Doors** — OfCategory(BuiltInCategory.OST_Doors).WhereElementIsNotElementType():
   - Count
   - DoorFamilies: distinct (e as FamilyInstance)?.Symbol?.Family?.Name, max 20

9. **Windows** — OfCategory(BuiltInCategory.OST_Windows).WhereElementIsNotElementType():
   - Same as doors

Cap any list extraction at 500 items to keep payloads reasonable. If a section overflows, take first 500 and note it (no need to flag in payload — just truncate).

public static class SnapshotClient
{
    private static readonly HttpClient _http = new HttpClient
    {
        Timeout = TimeSpan.FromSeconds(30)
    };

    private static readonly JsonSerializerOptions _jsonOpts = new()
    {
        PropertyNamingPolicy = JsonNamingPolicy.CamelCase
    };

    public static async Task<PostResult> PostAsync(SnapshotPayload payload, Settings settings)
    {
        var url = settings.ReplitUrl.TrimEnd('/') + "/api/snapshots";
        var json = JsonSerializer.Serialize(payload, _jsonOpts);
        var content = new StringContent(json, System.Text.Encoding.UTF8, "application/json");
        content.Headers.Add("x-snapshot-secret", settings.Secret);

        var response = await _http.PostAsync(url, content);
        if (!response.IsSuccessStatusCode)
        {
            var body = await response.Content.ReadAsStringAsync();
            throw new HttpRequestException(
                $"POST {url} failed: {(int)response.StatusCode} {response.StatusCode}. Body: {body}");
        }

        var resultJson = await response.Content.ReadAsStringAsync();
        var result = JsonSerializer.Deserialize<PostResult>(resultJson, _jsonOpts);
        if (result == null) throw new Exception("Empty response from server.");
        return result;
    }
}

NOTE: the Replit server may need a CORS/header tolerance update to accept x-snapshot-secret — verify that in testing. If it 401s with the secret correctly set, that's a server-side header config issue, not a client one.
