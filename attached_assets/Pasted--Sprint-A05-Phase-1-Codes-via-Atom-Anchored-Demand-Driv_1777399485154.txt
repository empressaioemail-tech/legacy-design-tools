types.ts:
  export interface CodeSource {
    id: string;                          // matches code_atom_sources.sourceName
    label: string;
    sourceType: "pdf" | "html_scrape" | "api" | "manual";
    licenseType: "public_record" | "subscribed" | "permitted_use";
    
    listToc(jurisdiction: string, codeBook: string, edition: string): Promise<TocEntry[]>;
    fetchSection(url: string, context: { jurisdiction, codeBook, edition }): Promise<AtomCandidate>;
  }

  export interface AtomCandidate {
    jurisdiction: string;
    codeBook: string;
    edition: string;
    sectionNumber: string;
    sectionTitle: string;
    parentNumbers: string[];
    fullPath: string;
    bodyText: string;
    sourceUrl: string;
    crossReferences: string[];           // other section numbers seen in body text
  }

  export interface TocEntry {
    jurisdiction: string;
    codeBook: string;
    edition: string;
    sectionNumber: string;
    sectionTitle: string;
    sectionUrl: string;
    parentNumbers: string[];
    fullPath: string;
  }

Grand County implementation (lib/codes-sources/src/grandCounty):

  GrandCountyDesignCriteriaSource implements CodeSource
  
  listToc():
    1. Fetch https://www.grandcountyutah.net/146/Design-Criteria
    2. Parse HTML (Cheerio) to find all PDF links
    3. For each PDF (limit to first 20 to be safe):
       - Download the PDF (cap 50MB per file)
       - Use pdf-parse to extract text per page
       - Run heuristic chunking: split on ALL-CAPS headers + numbered sub-sections (e.g., "1.0 GENERAL", "2.1 SCOPE")
       - Each chunk becomes a TocEntry: sectionNumber from matched header, sectionTitle from first line after header, sectionUrl points to PDF + page anchor
    4. Return TocEntry[]
  
  fetchSection(url):
    1. Parse PDF + page reference from URL
    2. Re-extract text for that section (cache PDF locally if seen recently)
    3. Build AtomCandidate
  
  Politeness:
    - Min 2-second delay between PDF downloads
    - Realistic Mozilla/5.0 User-Agent + Accept-Language: en-US
    - Cache PDFs locally during a single warm-up run

Municode implementation (lib/codes-sources/src/municode) — BUILT ONLY IF SECTION 0 PERMITTED:

  MunicodeSource implements CodeSource
  
  listToc():
    1. Fetch the codebook root page
    2. Parse navigation tree to enumerate all sections
    3. Each entry becomes a TocEntry — sectionUrl is the Municode URL for that section
    4. Return TocEntry[]
  
  fetchSection(url):
    1. Fetch the Municode section page
    2. Parse with Cheerio using selectors identified in Section 0.E
    3. Return AtomCandidate
  
  Politeness:
    - Honor crawl-delay from robots.txt; minimum 2 seconds + 0-2s jitter
    - Realistic Mozilla User-Agent, identifying string with contact email per RFC convention
    - Cap: 200 requests per UTC day per IP (configurable via MUNICODE_DAILY_REQUEST_CAP)
    - Single-thread fetch queue
  
  Error handling:
    - 4xx (other than 429) → unrecoverable, mark failure, do not retry
    - 429 → exponential backoff up to 3 retries
    - 5xx → exponential backoff up to 3 retries
    - HTTP timeout → 1 retry with longer timeout, then fail

  export function getApplicableCodeBooks(jurisdictionId) — returns config

atoms/fetch.ts:
  Looks up the source adapter from lib/codes-sources by sourceName.
  Calls source.fetchSection(url) to get the AtomCandidate.
  Calls atoms/store.upsertAtom() to persist.
  
  This decoupling means future sources (real Municode API, ICC API) plug in as new lib/codes-sources packages without changes here.

atoms/store.ts upsertAtom():
  1. Compute contentHash if bodyText present
  2. SELECT by (jurisdiction, codeBook, edition, sectionNumber, contentHash)
  3. If exists: return existing, no change
  4. Otherwise INSERT with embedding=NULL
  5. Trigger embedding async (fire-and-forget; embedding fills in within seconds)
  6. Return the atom

atoms/retrieve.ts retrieveForQuery():
  1. Get applicable code books for jurisdiction
  2. Embed the question
  3. Cosine similarity SELECT FROM code_atoms WHERE jurisdiction match AND embedding IS NOT NULL
     ORDER BY embedding <=> $queryEmbedding LIMIT topK
  4. Filter similarity > 0.4
  5. Return RetrievalResult[]
  6. If results.length < 3, enqueue a warm-up task to fetch more sections (demand-driven gathering signal)

warmup/enqueue.ts enqueueWarmupForJurisdiction(jurisdiction):
  For each applicable code book:
    1. Look up source config from jurisdictions.ts
    2. Call source.listToc(jurisdiction, codeBook, edition)
    3. For each TocEntry, INSERT INTO code_atom_fetch_queue if not already queued
  Returns total queued count

warmup/worker.ts startWarmupWorker():
  Singleton (in-memory mutex). Runs:
  1. SELECT one queue row WHERE not yet fetched/failed, oldest first
  2. atoms/fetch.ts to fetch and store
  3. On success: UPDATE row.fetchedAt, atomId; sleep 5s
  4. On UpCodesAuthError or similar (not applicable to Phase 1 sources but pattern stays): halt 1h, log clearly
  5. On other errors: UPDATE row.failedAt + reason, continue
  6. If queue empty: sleep 30s

prompt/format.ts:
  formatRetrievedAtomsForPrompt(atoms) returns the <reference_code_sections> block.
  For atoms where bodyTextRetained=true: include full text.
  For atoms where bodyTextRetained=false: include only section reference (sectionNumber, sectionTitle, fullPath, sourceUrl), NOT body text. Mark with "(reference only — see source URL for text)".
