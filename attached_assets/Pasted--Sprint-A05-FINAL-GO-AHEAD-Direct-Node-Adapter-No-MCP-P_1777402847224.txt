  lib/codes-sources/src/municode/

    ├── parser.ts         (response → AtomCandidate / TocEntry mapping)

client.ts requirements:
  - Base URL: https://api.municode.com
  - User-Agent: "Hauska-CodeAtoms/0.1 (+nick@hauska.io)"
    Override when MUNICODE_USER_AGENT env var is set
  - Accept: application/json
  - Timeout: 30 seconds per request (matches MCP server's behavior)
  - Connection reuse via single httpx-equivalent (use undici or node-fetch with agent)
  - Rate limiting: minimum 1.5 second delay between requests + 0-1 second random jitter
  - Daily request cap: 500 (env: MUNICODE_DAILY_REQUEST_CAP, default 500)
  - In-memory daily counter, resets at UTC midnight
  - Retry on 429 / 5xx: exponential backoff up to 3 retries (1s, 2s, 4s)
  - On 4xx (other than 429): no retry, throw MunicodeError with status + body
  - Single-thread fetch queue (p-queue with concurrency=1) to ensure rate limit holds across concurrent callers

endpoints.ts (centralized, easy to update if API changes):
  GET /States/abbr?stateAbbr={code}              → state info
