SECTION 6 — BACKEND ROUTES (artifacts/api-server/src/routes/engagements.ts)
============================================================

Implement the two new endpoints.

PATCH /api/engagements/:id:
  1. Validate body via generated Zod schema; allow partial.
  2. SELECT engagement; 404 if missing.
  3. Build update set: only fields present in the body.
  4. If body.address is present and !== current.address:
     - Call geocodeAddress(body.address)
     - On success: include latitude, longitude, geocodedAt=now(), geocodeSource="nominatim", jurisdictionCity/State/Fips from response, siteContextRaw=full geocoder response in update set.
     - On null result or error: include only the address change; leave geo fields untouched, add a warning to a response.warnings array.
  5. UPDATE in a transaction with updatedAt=now().
  6. Re-SELECT and return EngagementDetail (including computed `site` field — see assembly note below).

POST /api/engagements/:id/geocode:
  1. SELECT engagement; 404 if missing; 400 if address is null.
  2. Call geocodeAddress(engagement.address). Update geo fields on success; on null/error, do not modify any rows but return 200 with a warning.
  3. Return EngagementDetail.

Site assembly: in the GET /api/engagements and GET /api/engagements/:id handlers, also build the `site` object:
  site = {
    address: engagement.address,
    geocode: (engagement.latitude !== null) ? {
      latitude, longitude, jurisdictionCity, jurisdictionState, jurisdictionFips,
      source: engagement.geocodeSource, geocodedAt: engagement.geocodedAt
    } : null,
    projectType: engagement.projectType,
    zoningCode: engagement.zoningCode,
    lotAreaSqft: engagement.lotAreaSqft
  }

Use @workspace/site-context's geocodeAddress, NOT a fresh Nominatim call inline. This keeps rate-limiting centralized.

Reminder: the C# Revit add-in's POST /api/snapshots payload includes an `address` field from Revit's projectInformation. When auto-creating an engagement, ALSO copy that address onto the new engagement (so the user starts with a populated address) AND geocode it (best-effort, swallow errors). This means new engagements arrive pre-geocoded when Revit gave us an address.
