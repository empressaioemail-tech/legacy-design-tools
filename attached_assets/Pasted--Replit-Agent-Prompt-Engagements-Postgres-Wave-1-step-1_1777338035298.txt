SECTION 1 — DATABASE SCHEMA (lib/db/src/schema/)
============================================================

Define two tables. Split into engagements.ts and snapshots.ts for clarity, then re-export from schema/index.ts.

engagements
  id            uuid PK, default random
  name          text not null            (matches the Revit projectName as captured)
  nameLower     text not null UNIQUE     (case-insensitive lookup key)
  jurisdiction  text nullable            (e.g. "Moab, UT")
  address       text nullable
  status        text not null default "active"   (active | on_hold | archived)
  createdAt     timestamptz default now() not null
  updatedAt     timestamptz default now() not null
  index on nameLower

snapshots
  id            uuid PK, default random
  engagementId  uuid not null, FK → engagements.id ON DELETE CASCADE
  projectName   text not null            (preserved per-snapshot in case engagement renames)
  payload       jsonb not null           (full Revit snapshot)
  sheetCount    integer nullable
  roomCount     integer nullable
  levelCount    integer nullable
  wallCount     integer nullable
  receivedAt    timestamptz default now() not null
  index on engagementId
  index on receivedAt

Define Drizzle relations between them. Export TS types via $inferSelect / $inferInsert from each file.

SECTION 4 — BACKEND ROUTES (artifacts/api-server/src/routes/)
============================================================

Replace the in-memory Map entirely. Use the generated Zod schemas to validate request bodies/params.

NEW: routes/engagements.ts

  GET /api/engagements
    Drizzle query that returns each engagement plus:
      - snapshotCount (subquery COUNT)
      - latestSnapshot (subquery LIMIT 1 ordered by receivedAt desc, fields per SnapshotSummary)
    Order by engagements.updatedAt DESC.
    N+1 is fine for now (we have <50 engagements). Optimize when it matters.

  GET /api/engagements/:id
    Validate id as uuid. 404 if not found.
    Return EngagementDetail: the engagement row + snapshots array (SnapshotSummary, newest first, no payload).

REWRITE: routes/snapshots.ts

  POST /api/snapshots:
    1. Validate x-snapshot-secret header → 401 if mismatch.
    2. Parse body (loose) — require projectName: string.
    3. Compute nameLower = projectName.trim().toLowerCase()
    4. WITHIN A TRANSACTION:
       a. SELECT engagement WHERE nameLower = $1
       b. If absent: INSERT new engagement (name=projectName, nameLower, status=active, address=null, jurisdiction=null). Capture the new row.
       c. INSERT snapshot:
          - engagementId
          - projectName (from body)
          - payload (full body)
          - sheetCount = body.sheets?.length ?? null
          - roomCount  = body.rooms?.length ?? null
          - levelCount = body.levels?.length ?? null
          - wallCount  = body.walls?.count ?? body.walls?.length ?? null   ← handle both shapes; the C# sender uses { count, totalLengthFeet, wallTypes }
       d. UPDATE engagements SET updatedAt = now() WHERE id = engagementId
    5. Return { id, receivedAt, engagementId, engagementName, autoCreated }

  GET /api/snapshots:
    Return flat list joined to engagement so each row has engagementName. Newest first. SnapshotSummary shape + engagementName.

  GET /api/snapshots/:id:
    Return full SnapshotDetail including payload. 404 if not found.

  DELETE the exported in-memory getSnapshot() helper. Anything that imported it (chat.ts) must now query the DB.

REWRITE: routes/chat.ts

  POST /api/chat:
    1. Validate body via generated Zod schema → { engagementId, question, history? }
    2. SELECT engagement; 404 if missing.
    3. SELECT latest snapshot for this engagement (LIMIT 1 by receivedAt desc); 400 with code "no_snapshots" if none.
    4. Build system prompt:
       "You are helping an architect understand their Revit model for the engagement '{name}'{address ? ' at ' + address : ''}{jurisdiction ? ' (' + jurisdiction + ')' : ''}. The most recent snapshot was captured {relative-time-string}.\n\nAnswer grounded in the snapshot data below. If the data does not contain what's asked, say so plainly. Be terse and operational in tone — this is a professional tool, not a chatbot.\n\n<snapshot received_at='{ISO}'>\n{JSON.stringify(payload, null, 2)}\n</snapshot>"
    5. Stream Claude response via @workspace/integrations-anthropic-ai (model claude-sonnet-4-6, max_tokens 1024, stream: true) with the existing SSE pattern. Don't change the SSE event format.
